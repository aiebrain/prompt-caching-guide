프롬프트 캐싱이란?

이전에 처리한 프롬프트 내용을 저장해두고, 다음 요청에서 재사용해서 비용과 시간을 줄이는 기술입니다.

🏢 직원 비유로 이해하기
방식예시결과
캐싱 없음매번 새 직원에게 회사 규정 100페이지 읽히기❌ 매번 시간/비용 소모
캐싱 있음"규정은 이미 읽었어. 오늘 할 일만 말해줘"✅ 처음 1번만 읽음
핵심 원칙: 고정 내용(역할·규칙)은 앞에, 변동 내용(날짜·질문)은 뒤에 배치하면 캐시 히트율이 극대화됩니다.

FAQ

자주 묻는 질문

Q. 코드를 직접 수정해야 하나요?
아니요. 2026년 Anthropic 자동 캐싱으로 아무것도 안 해도 됩니다. 단, 프롬프트 구조를 최적화하면 캐시 히트율이 높아져 더 많이 절감할 수 있습니다.
Q. 캐시가 효과 있는지 어떻게 알 수 있나요?
Claude API 응답에 cache_read_input_tokens 값이 있으면 캐시 히트된 것입니다. 이 값이 클수록 더 많이 절감되었다는 의미입니다.
Q. 언제 캐싱이 특히 효과적인가요?
① 매일 반복되는 크론잡 ② 동일한 시스템 프롬프트를 쓰는 앱 ③ CLAUDE.md를 활용하는 Claude Code 프로젝트 ④ 긴 문서를 반복 분석하는 작업
Q. 캐시 유효 시간은 얼마인가요?
기본 5분입니다. 설정 시 최대 1시간까지 연장 가능합니다. 5분이 지나면 다시 캐시가 만들어집니다.
Q. 다른 사용자와 캐시가 공유되나요?
아니요. 캐시는 같은 API 키 내에서만 공유됩니다. 보안 문제 없습니다.
